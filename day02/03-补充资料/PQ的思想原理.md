# PQ的思想原理

## 一句话理解什么是 Product Quantization：

> 将一个高维向量拆成多个子向量，对每个子向量分别进行量化编码，从而用较少的存储代价表示原始向量，同时可以快速近似计算向量间的距离。

------

## 🔍 背景场景

我们经常在做如下事情：

- 有一个超大的向量数据库（如图像特征、文本向量），每个向量维度可能是 128、256 或 1024；
- 现在有一个查询向量，我们要在数据库中找最相似的向量（比如 top-10 相似）；
- 向量越多、维度越高，查找越慢、内存越爆炸 💥。

### 🧩 解决问题的目标：

- **压缩**每个高维向量，用更少的比特表示；
- **近似计算距离**（如欧氏距离），比暴力计算更快；
- 同时保持比较高的**精度**。

这时候就可以使用——Product Quantization（PQ）！

------

## ✳️ 核心思想

将一个高维向量拆成多个低维“子向量”，然后对每个子向量分别做 **K-Means 聚类**，得到一个“编码表”，每个子向量就用它最接近的聚类中心的编号（索引）来表示。

这个过程叫“量化”，所以是“乘积量化”——在多个子空间分别量化，然后组合起来。

------

## 🧮 数学原理讲解

### 假设：

- 原始向量维度为：`D = 128`
- 我们把它划分为 `M = 8` 个子向量段，每个子向量维度是 `D/M = 16`
- 每个子空间分别做 `k` 个聚类（比如 `k=256`）

------

### 🎯 步骤分解：

#### 第 1 步：向量分块（Subspace Division）

把一个原始向量 `x ∈ ℝ^128` 拆成 `M=8` 个子向量：

```
x → [x₁, x₂, ..., x₈]   每个 xᵢ ∈ ℝ^16
```

------

#### 第 2 步：每个子向量空间独立做 K-means 聚类

对第 i 个子空间，做 `k=256` 的聚类，得到 `256` 个聚类中心（codewords）：

```
Cᵢ = {cᵢ¹, cᵢ², ..., cᵢ²⁵⁶}    其中 cᵢⱼ ∈ ℝ^16
```

------

#### 第 3 步：编码每个子向量（Vector Quantization）

对于 `xᵢ`，我们找到它在第 i 个子空间中最接近的聚类中心索引：

```
qᵢ = argminⱼ ||xᵢ - cᵢⱼ||²
```

所以整个向量 `x` 最终被编码为：

```
[q₁, q₂, ..., q₈]   每个 qᵢ ∈ {0, ..., 255}
```

这样，一个原始向量就被压缩成 `8 字节`（如果每个索引用 1 字节表示）。

------

## 💡 为什么叫 Product Quantization？

因为我们从多个独立子空间中分别编码，每个子空间有 `k=256` 个码本，所以总共可以表示的组合是：

```
256^8 = 巨大数量
```

这种组合构成的整体编码空间就是原空间的一个“乘积空间”（笛卡尔积），所以叫 **乘积量化**。

------

## ⚡ 快速距离计算

压缩之后，如何计算距离呢？

假设你有一个查询向量 `q`，你可以：

1. **不对查询向量做量化**，仍然保持为原始浮点向量；
2. 把它拆成子向量 `[q₁, ..., q₈]`；
3. 对于数据库中某个编码 `[x₁, ..., x₈]`，我们直接查表计算距离：

```
Dist(q, x) ≈ ∑_{i=1}^{8} ||qᵢ - cᵢ^{xᵢ}||²
```

即：对每段，查一下当前编码的聚类中心，然后与查询向量那段的欧式距离，8次查表+加法，极快！
