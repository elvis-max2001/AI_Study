# TF-IDF和BM25的区别和联系

## 🌟背景场景：

假设我们有以下三份文档（Corpus）：

```properties
Doc1: 我 喜欢 吃 苹果  
Doc2: 我 今天 吃 了 一个 苹果  
Doc3: 苹果 是 一种 水果  啊
```

现在你搜索的关键词是：

```properties
Query: 吃 苹果
```

我们要用 **TF-IDF** 和 **BM25** 分别计算每个文档与查询的相关性打分。

------

## 1️⃣ TF-IDF 原理 + 示例

**TF-IDF 核心思想：**

- **TF（Term Frequency）**：某词在文档中出现得越多，该词对该文档越重要。
- **IDF（Inverse Document Frequency）**：某词在所有文档中越少见，区分度越高，重要性越高。

**公式：**

![image-20250730115546060](img/01.png)

其中：

- t：词语
- d：文档
- N：文档总数（3）
- DF(t)：包含词 t的文档数

------

**示例：计算“吃 苹果”在每个文档中的得分**

### ➤ 第一步：统计词频（TF）

| 词语 | Doc1 | Doc2 | Doc3 |
| ---- | ---- | ---- | ---- |
| 吃   | 1    | 1    | 0    |
| 苹果 | 1    | 1    | 1    |



### ➤ 第二步：计算 DF 和 IDF

| 词语 | DF（包含它的文档数） | IDF                           |
| ---- | -------------------- | ----------------------------- |
| 吃   | 2                    | log(3/2 + 1) ≈ 0.405          |
| 苹果 | 3                    | log(3/3 + 1) = log(2) ≈ 0.693 |



### ➤ 第三步：TF-IDF 得分（简单相加）

| 文档 | 吃的TF-IDF | 苹果的TF-IDF | 总分  |
| ---- | ---------- | ------------ | ----- |
| Doc1 | 1 × 0.405  | 1 × 0.693    | 1.098 |
| Doc2 | 1 × 0.405  | 1 × 0.693    | 1.098 |
| Doc3 | 0 × 0.405  | 1 × 0.693    | 0.693 |



------

## 2️⃣ BM25 原理 + 示例

**BM25 核心思想：**
 比 TF-IDF 更复杂，考虑了文档长度、词频饱和、查询词的重要性。用于计算查询（Query）与文档（Document）之间的相关性得分

**公式**

![image-20250730115821172](img/02.png)

默认参数：

- fi 是词项 qi在文档 D中的出现次数（TF），
- N是文档总数，
- ni是包含词项 qi的文档数，
- ∣D∣是文档 D的长度，
- avgdl 是所有文档的平均长度，
- k1 和 b是调整参数，通常设置为 k1=1.2和 b=0.75 。

------

### ➤ 示例参数：

- Doc1 长度 = 4
- Doc2 长度 = 6
- Doc3 长度 = 5
- 平均长度 avgdl = (4+6+5)/3 = 5

用 BM25 公式代入计算（略去繁琐步骤，给出简化结果）：

| 文档 | BM25 得分（吃 + 苹果） |
| ---- | ---------------------- |
| Doc1 | ≈ 1.45                 |
| Doc2 | ≈ 1.52  ✅（最高相关）  |
| Doc3 | ≈ 0.68                 |



> 由于 Doc2 更长、包含更多上下文信息，它在 BM25 中被认为更相关。

------

## ✅总结对比：

| 特性             | TF-IDF     | BM25（改进版）             |
| ---------------- | ---------- | -------------------------- |
| 是否考虑文档长度 | ❌ 不考虑   | ✅ 考虑（更公平）           |
| 词频处理         | 线性累加   | 非线性饱和处理（防止刷词） |
| 效果             | 中等，快速 | 更精确，现代搜索引擎常用   |
| 是否使用参数     | 无         | 有 k1,b参数可调节          |
