# 加权排名策略（WeightedRanker）

加权排名策略（WeightedRanker）是一种融合多个向量检索结果评分（score）的方法，核心是根据不同向量字段的重要性，为各路径搜索结果的得分分配权重，再将这些加权得分融合，得出最终排序。

## WeightedRanker 原理和步骤

1. **收集搜索得分**
   从不同的向量检索路径分别收集返回的结果及其得分（例如 score_1、score_2）。

2. **得分归一化**
   不同路径的得分范围通常不同（如内积 IP 可能是 [-∞,+∞]，欧氏距离 L2 是 [0,+∞]），直接比较不合适。
   通过对分数使用如函数等数学变换，将各得分统一映射到  区间，方便统一秩序比较。

3. **分配权重**
   根据不同向量字段的重要性，为每一路径的归一化得分分配权重 wi*，权重值在0-1之间。
   例如图像特征权重 0.6，文本特征权重 0.4。

4. **融合得分**
   按权重加权计算得分：

   $score^{final}$=$∑w^{i} *score^{i}$

   最后根据加权融合的得分从高到低排序，得到最终搜索结果的优先级。

## 举个具体例子（多模态搜索）

假设有两路检索：

| 文档ID | 图像检索得分 | 文本检索得分 |
| :----- | :----------- | :----------- |
| 101    | 0.92         | 0.87         |
| 203    | 0.88         | N/A          |
| 150    | 0.85         | N/A          |
| 198    | 0.83         | 0.91         |
| 175    | 0.80         | 0.82         |
| 110    | N/A          | 0.85         |
| 250    | N/A          | 0.78         |

假设权重为图像 0.6，文本 0.4，则加权得分计算为：

| 文档ID | 最终得分 = 图像得分×0.6 + 文本得分×0.4     |
| :----- | :----------------------------------------- |
| 101    | 0.92×0.6 + 0.87×0.4 = 0.552 + 0.348 = 0.90 |
| 203    | 0.88×0.6 + 0 = 0.528                       |
| 150    | 0.85×0.6 + 0 = 0.51                        |
| 198    | 0.83×0.6 + 0.91×0.4 = 0.498 + 0.364 = 0.86 |
| 175    | 0.80×0.6 + 0.82×0.4 = 0.48 + 0.328 = 0.81  |
| 110    | 0 + 0.85×0.4 = 0.34                        |
| 250    | 0 + 0.78×0.4 = 0.312                       |

最终排序就是：

101 (0.90) > 198 (0.86) > 175 (0.81) > 203 (0.528) > 150 (0.51) > 110 (0.34)> 250(0.312)

## 关键点总结

- WeightedRanker 适合多模态或多特征联合搜索，能体现不同特征的重要性和贡献度。
- 归一化步骤很重要，确保不同度量标准的得分能统一比较。
- 权重由用户设定，灵活调整各检测路径的优先级。
- 最终按加权得分降序排序，得到综合考虑所有搜索路径结果的最优排序。
