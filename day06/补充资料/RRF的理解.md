# RRF的理解

倒数排序融合（Reciprocal Rank Fusion，简称 RRF）是一种**融合多个独立排名列表**的方法，目的是合并多个检索器（搜索系统）返回的结果，使最终结果能较好地反映所有检索器的优势，提高整体相关性和鲁棒性。

## RRF 的核心思想和公式

RRF 利用**排名位置的倒数加权累加**来给每个文档打分

**关键点：**

- 排名越靠前，倒数越大，贡献分数越高。
- k 加上排名，有平滑作用，避免前几名分数差距过大。
- 出现在多个排名列表且位置靠前的文档，累积得分更高，最终排名优先。

## 通俗举例说明

假设有2个检索器，它们分别给出以下两个排名结果列表（排名从1开始）：

| 文档ID | 检索器1排名 | 检索器2排名 |
| :----- | :---------- | :---------- |
| D1     | 1           | 4           |
| D2     | 2           | 1           |
| D3     | 3           | 6           |
| D4     | — (无排名)  | 2           |
| D5     | 4           | — (无排名)  |

取 k=60

| 文档ID | 检索器1得分           | 检索器2得分           | RRF总分 |
| :----- | :-------------------- | :-------------------- | :------ |
| D1     | 1/（60+1+1）= 0.01639 | 1/（60+4+1）= 0.01563 | 0.03202 |
| D2     | 1/（60+2+1）= 0.01613 | 1（60+1+1）= 0.01639  | 0.03252 |
| D3     | 1/（60+3+1）= 0.01587 | 1（60+6+1）= 0.01538  | 0.03125 |
| D4     | 0 (无排名，因此得分0) | 1（60+2+1）= 0.01613  | 0.01613 |
| D5     | 1/（60+4+1）= 0.01563 | 0                     | 0.01563 |



最终根据 RRF 总分排序，得到的融合排名是：

D2(0.03252)>D1(0.03202)>D3(0.03125)>D4(0.01613)>D5(0.01563)

## 公式运算步骤总结

1. 对每个检索器，第 i 个结果中排名为 ranki(d) 的文档 d，计算分数 1/(k+ranki(d)+1)。
2. 若文档未出现在某检索器结果中，则贡献分数为0。
3. 将所有检索器对应的分数相加，得到文档 d 的总 RRF 分数。
4. 根据总分高低对所有文档重新排序。
